# 報告同步改進 - 已知問題集成

## 📊 報告內容更新

已將已知問題系統與報告生成完全同步。

### 1. 報告頭部統計

報告現在在標題下方展示：
```
**已知問題**: 8 | **新問題**: 4
```

這讓工程師一眼看到已知 vs 新問題的比例。

### 2. 頂級問題詳細信息

對於前 5 個高優先級問題，現在包含：

```markdown
**已知問題**: `ISSUE-001` - 索引不匹配錯誤
**分類**: logic
```

如果是已知問題，會顯示：
- Issue ID（唯一標識碼）
- 問題名稱（中文描述）
- 分類（logic, parsing, infrastructure 等）

### 3. 其他問題表格

低頻率問題表格增加了狀態欄：

```
| 問題名稱 | 位置 | 發生次數 | 狀態 | 嚴重性 |
|---------|------|--------|------|-------|
| player not found | ... | 247 | ✅ ISSUE-006 | high |
| game config error | ... | 4 | 🆕 新問題 | low |
```

狀態欄顯示：
- `✅ ISSUE-XXX` - 已知問題（帶 Issue ID）
- `🆕 新問題` - 未識別的新問題

## 🔄 工作流程

```
錯誤群組分析
  ↓
已知問題匹配
  ↓
設定 IsKnown flag 和 IssueID
  ↓
報告生成
  ↓
顯示完整的已知/新問題信息
```

## 🎯 優勢

✅ **一目了然** - 立即看到哪些是已知問題  
✅ **便於追蹤** - Issue ID 方便尋找相關文檔或工單  
✅ **優先級更清晰** - 新問題 (🆕) 優先於已知問題 (✅)  
✅ **自動化** - 無需手動標記，系統自動識別  
✅ **中文友善** - 所有文字完全本地化

## 🚀 使用

現在運行分析時：

```bash
go run cmd/analyzer/main.go -fetch -time 24h -output ./reports
```

生成的報告會自動包含已知問題信息！

沒有任何額外的配置需要。已知問題匹配在後台自動進行。
